## 1. 当前项目定位

这是一个围绕 LifeUp 的电脑端管理工具，目标不是替代手机 App，而是解决手机端批量配置效率低的问题：

- 本地备份模式：读取 LifeUp 备份 ZIP，编辑 SQLite 工作副本，再导出备份。
- 手机云人升模式：读取手机实时数据，当前安全边界是只读已有数据，已确认的写入方向主要是新增任务。
- 长期目标控制台：首页优先回答长期目标、成就进度、停滞目标和奖励体系是否支撑成长。
- 管理页仍保留：任务、商品、成就、背包、合成、卡池、经济等作为控制台下钻页面。

最重要的产品原则：统一入口，分离数据源。不要把手机实时数据自动写进本地备份，也不要把本地备份误写到手机。

## 6. 安全边界

请继续遵守：

- 原始备份不要随便覆盖。
- 本地备份写入应走工作副本、保存、导出备份流程。
- 云人升模式不做自动同步。
- 云端已有任务、商品、成就当前只读。
- 当前不要做手机端已有数据编辑或删除。
- 不把云端数据写入本地备份。
- 不调用 `/data/import` 作为偷懒方案。
- 不直接写手机端 SQLite。
- 云端写操作必须先预览、再确认。
- `lifeup_cloud_config.json` 只保存 Host 和端口，不保存 Token。

### 7.1 README 过时

`README.md` 仍描述旧版“无需手机 API”的定位。下个 agent 如果继续整理文档，建议优先更新 README：

### 7.2 番茄桥接仍需真机校准

当前实现读取 `/pomodoro_records`，但不同 LifeUp 版本的记录字段可能不同。

建议下一步：

- 在手机端完成一次带 `闭关修炼` 标签的番茄。
- 读取 `POST /api/cloud/data` 中的 `pomodoro_records` 原始数据。
- 校准 `normalize_focus_record()` 的字段映射。

### 7.3 商品效果需要回归测试

新增商品弹窗已经很复杂，尤其是商品效果、限制、开箱、合成配方的组合。

建议重点测试：

- 新建普通商品。
- 新建带购买限制的商品。
- 新建带金币/经验/商品数量效果的商品。
- 新建开箱商品。
- 新建简易合成商品，确认 `synthesismodel` 和 `synthesisconnmodel` 写入正确。
- 编辑已有商品不应误删原复杂效果。

### 7.4 云端连接不稳定是常态

手机云人升依赖：

- 手机和电脑同局域网。
- 手机云人升服务开启。
- 手机 IP 和端口正确。
- 手机系统没有杀后台。

UI 必须允许云端失败，并能清楚回退到本地备份。

### 7.5 当前端口与旧文档不一致

旧交接档写的是 `5000`，当前运行服务是 `5001`。后续写文档或测试时不要照抄旧端口，先检查：
