# LifeUp Dashboard 用户指南

本指南面向第一次使用的用户。最重要的一句话是：Dashboard 修改的是工作副本，不会直接覆盖你选中的原始备份。

## 1. 选择运行方式

### 浏览器版

适合开发和检查功能。先运行 `python server.py`，再访问 `http://127.0.0.1:5000/`。

不要直接双击 `index.html`。浏览器地址如果以 `file://` 开头，文件选择和加载接口不能正常工作。

### 桌面版

适合日常使用。运行 `LifeUpDashboard.exe` 后会打开独立窗口。桌面版的本机数据位于：

```text
%LOCALAPPDATA%\LifeUpDashboard
```

这个目录可能包含配置、工作副本、快照、导出和日志，因此不要把整个目录当作发布包分享给别人。

## 2. 本地备份模式

### 打开备份

1. 在 LifeUp App 中导出 ZIP 备份。
2. 保留这个原始 ZIP，不要用新文件覆盖它。
3. 在 Dashboard 顶部确认数据源是“本地备份”。
4. 点击“打开存档”→“选择文件”。
5. 页面显示“已加载”后再开始浏览或修改。

浏览器版会把上传文件复制到 `workspaces/browser-imports/`；桌面版直接读取你选中的来源 ZIP，但所有数据库修改仍只发生在临时解压工作区。

### 修改前创建快照

1. 打开侧栏“快照管理”。
2. 点击“创建快照”。
3. 输入容易识别的名称，例如“批量改任务前”。
4. 等待页面显示归档和数据库检查均正常。

快照是恢复点，不属于普通临时文件，“本机维护”页不会自动列出或删除快照。

### 批量导入或修改

任务、商品、自定义成就和图标操作都应先看预览：

1. 下载页面提供的 CSV/JSON 模板。
2. 填写后上传文件。
3. 检查每一行的动作、重复提示和错误字段。
4. 有错误时先修改文件并重新预览。
5. 确认无误后才执行。服务端会自动创建快照并在一个事务中完成写入。

### 安全导出

1. 点击顶部“导出备份”。
2. 等待页面报告 ZIP、SQLite 和关键表数量检查通过。
3. 记下新文件的位置。浏览器开发版默认在项目 `exports/`，桌面版默认在 `%LOCALAPPDATA%\LifeUpDashboard\exports`。
4. 用 Dashboard 再次打开新 ZIP，核对关键数据。
5. 只有核对无误后，才在手机 LifeUp 中手动恢复新 ZIP。

导出失败时，原始备份和当前来源文件不会被覆盖。请检查磁盘空间、目录权限和文件是否被其他程序占用。

## 3. 手机云人升模式

### 连接

1. 切换数据源到“手机云人升”。
2. 打开“手机连接/新增任务”。
3. 填写手机地址、端口和 Token。
4. 点击连接测试，按页面提示处理配置、网络、认证、超时或响应格式问题。

Token 只保存在当前服务进程内存中。关闭服务后需要重新输入；操作记录、配置文件和发布包都不会保存 Token。

### 读取与新增任务

- 手机是云端实时数据的事实来源。
- 已有任务、商品和成就只读，不支持编辑或删除。
- 新增任务会实时读取手机分类和技能。
- 单条或 CSV/JSON 批量新增都必须先预览。
- “确认新增”只在预览有效时启用，并通过幂等键避免重复执行。
- 网络超时造成结果不确定时，不要立即重试；先刷新手机任务列表确认。

## 4. 本机维护

“本机维护”页用于删除项目托管的旧导入副本、旧恢复副本、自动导出和 `work/` 临时文件。

操作步骤：

1. 点击“扫描可清理文件”。此时只生成预览，不删除文件。
2. 查看文件类型、名称、大小、修改时间和识别依据。
3. 只勾选你确定不再需要的项目。
4. 点击“删除选中项”。
5. 在二次确认窗口核对数量和大小，再点击确认。

安全限制：

- 原始备份、当前工作副本、快照、手机数据和本机配置不会进入列表。
- 不符合项目命名规则的普通文件不会进入列表。
- 预览 10 分钟后失效，且只能执行一次。
- 文件在预览后发生变化时，服务端会拒绝删除该文件。

## 5. 常见问题

### 点击“选择文件”没有弹窗

先看浏览器地址。若以 `file://` 开头，请关闭该页面，运行 `python server.py`，然后访问 `http://127.0.0.1:5000/`。

### 手动路径加载失败

浏览器不能替你绕过 Windows 权限。最稳定的方式是点击“选择文件”，让 Dashboard 上传并创建托管工作副本。

### 新建任务失败

- 本地模式：确认已经加载备份，并检查标题、分类和数字字段。
- 云端模式：重新测试手机连接，确认预览尚未过期，并查看逐条结果。
- 显示“待确认”时，先去手机核对，不要连续点击执行。

### 页面显示旧内容

确认运行的是当前项目的 `server.py` 或新构建的 `LifeUpDashboard.exe`。`dist/` 中的历史 EXE 不会自动跟随源码更新。

## 6. 需要备份哪些文件

- 必须长期保留：你从 LifeUp 导出的原始 ZIP，以及经过核对的新导出 ZIP。
- 建议保留：重要修改前创建的快照。
- 可按需清理：旧浏览器导入副本、旧恢复副本、已确认不需要的自动导出和临时日志。
- 不要分享：Token、桌面配置、工作副本、快照、日志或任何包含个人 LifeUp 数据的数据库文件。

## 7. 使用 Codex 或 Claude 查询 LifeUp

项目提供 `mcp_server.py`，让支持 MCP 的 AI 客户端通过正在运行的 Dashboard 查询数据。它不会直接打开备份、数据库或手机配置。

最重要的规则：

- 每次查询都要明确选择 `local` 或 `cloud`，失败时不自动换数据源。
- MCP 没有编辑、删除或批量写入工具。
- 手机 Token 只在 Dashboard 进程内存中，不要放进 Codex/Claude 配置。
- 新增手机任务必须先预览，核对任务摘要后由你明确确认，并使用幂等键执行。
- 网络超时或结果不确定时，先查看手机任务列表，不要立即重试。

完整安装命令、客户端配置和工具说明见 [LifeUp Dashboard MCP 使用指南](MCP_GUIDE.md)。
