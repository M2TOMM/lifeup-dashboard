# ADR-003：云端错误分类、分类级渐进加载与进程内缓存

## 状态

已接受（2026-07-18）

## 背景

云人升连接可能因为 Host、端口、局域网、Token、响应时间或 API 格式而失败。Task 13 之前，这些问题大多被压成普通异常字符串；成就读取还需要先取分类，再逐类读取，任一慢接口都会让用户长时间只看到一个整体加载状态。

本项目同时有两个不可突破的边界：手机是云端实时数据的事实来源，失败时不能回退或混入本地备份；Token 只能存在服务进程内存中，不能进入磁盘配置、日志或缓存。

## 决策

### 1. 使用稳定错误分类

云端连接错误统一返回 `code`、`category`、`error`、`suggestion` 和 `retryable`：

- `CLOUD_HOST_MISSING`：缺少 Host。
- `CLOUD_PORT_INVALID`：端口不是 1～65535 的整数。
- `CLOUD_CONNECTION_REFUSED`：目标拒绝连接，通常是服务未开启或端口错误。
- `CLOUD_NETWORK_UNREACHABLE`：其他局域网或地址不可达问题。
- `CLOUD_TIMEOUT`：只读请求超时；写入超时仍提示先去手机确认，不鼓励立即重试。
- `CLOUD_AUTH_FAILED`：HTTP/API 认证失败。
- `CLOUD_RESPONSE_INVALID`：不是 JSON、不是对象或缺少 API 状态码。
- `CLOUD_HTTP_ERROR` / `CLOUD_API_ERROR`：其他 HTTP 或 API 响应错误。

页面按 `category` 显示初学者可理解的标题，并同时保留稳定错误码供测试和排查。

### 2. 成就按分类独立加载

前端先读取 `/achievement_categories`，立即列出全部分类；随后最多并行读取 3 个 `/achievements/{id}`。每个分类单独维护 `pending`、`loading`、`success` 或 `error` 状态。

单类失败不会清空其他分类的成功结果。失败分类提供“只重试此分类”，重试使用 `force_refresh=true`，不会重新请求或覆盖已经成功的分类。

`POST /api/cloud/data` 保留原有 `dataset=achievements` 聚合契约，并新增 `dataset=achievement_category` 供渐进页面逐类请求。

### 3. 只读响应使用 30 秒进程内缓存

缓存键只包含 `base_url` 和只读路由，缓存值只包含响应、抓取时间和过期时间。Token 值不进入缓存键或缓存值。

- 用户显式提交 Token 的请求绕过缓存。
- 内存 Token 新增、变化或清除时清空全部云端读取缓存。
- 页面显示“手机实时读取”或“内存缓存”以及实际抓取时间。
- “忽略缓存刷新全部”和单类重试都显式使用 `force_refresh=true`。
- 服务退出或重启后缓存自然消失，不写入任何文件。

## 备选方案

### 把缓存写入本地文件

拒绝。持久化会让过期手机数据跨进程继续存在，也增加 Token 或私人数据误落盘的风险。

### 成就继续由一个聚合请求整体返回

保留为兼容接口，但不作为页面主流程。整体请求无法及时展示进度，也无法在不清空成功数据的前提下只重试一个分类。

### 云端失败时展示本地备份数据

拒绝。这样会让用户把旧备份误认为手机实时状态，违反 local/cloud 严格隔离。

## 结果与影响

- 用户可以根据错误类型采取具体操作，不再反复猜测 Host、端口或 Token。
- 慢分类和失败分类不会阻塞或清空成功分类。
- 30 秒缓存减少短时间重复读取手机的压力，同时来源和抓取时间保持可见。
- 缓存不是同步或离线模式；过期或强制刷新失败时仍明确报错，不回退本地数据。
- 云端写入安全流程、预览令牌和幂等执行逻辑保持不变。
