# ADR-002：真实日常与番茄热力图的兼容口径

## 状态

Accepted

## 日期

2026-07-18

## 背景

Task 13 要把“日常功课”的随机月度热力图替换为真实任务完成和番茄记录，并让“闭关修炼”也能查看同一口径的真实番茄热力图。不同 LifeUp 版本和本地/云端接口使用的时间、时长字段并不完全相同；当前真实备份还没有三大功课分类和番茄记录，因此不能靠界面假数据补齐。

## 决策

### 数据范围与隔离

- `GET /api/activity/heatmap?source=local|cloud&period=day|week|month` 是热力图的唯一数据入口。
- 任务热力图统计全部真实已完成任务，不限制为“筑基日常”“试炼悬赏”“位面征服”三个界面分类；这三个分类只影响日常页面的任务列表。
- 番茄热力图统计全部未删除、未放弃且时长大于 0 的真实番茄记录。“闭关修炼”页面原有的标签筛选仍只影响闭关记录列表和旧概览卡片。
- `local` 只读取当前工作副本 SQLite；`cloud` 只调用手机只读 `/tasks` 和 `/pomodoro_records`，失败时不回退本地。

### 时间字段优先级

- 本地任务：`updatedtime` → `endtime` → `createdtime`。
- 云端任务：`finishedTime` / `finishTime` / `completedTime` / `completionTime` → 更新时间 → 结束时间。
- 番茄：开始时间 → 结束时间 → 创建时间。
- Unix 秒、Unix 毫秒和 ISO 8601 均可识别；带时区的 ISO 时间先换算为运行 Dashboard 的电脑本地时区，再按日期分桶。
- 字段不存在或值无法识别时跳过该条记录并显示缺口，不使用当前状态猜测历史日期。

### 番茄时长

- `durationMinutes`、`duration_min`、`minutes` 等分钟字段直接按分钟处理。
- `lastTime` / `lasttime` 等值大于 600 时按毫秒换算为分钟，这是现有本地数据库的实际存储方式。
- 没有可用时长时，只有开始和结束时间都有效才按两者差值计算；否则跳过并提示。
- 删除和放弃的番茄不计入。

### 时间范围与连续天数

- 今日：电脑本地日期 00:00 至次日 00:00。
- 本周：周一 00:00 至下周一 00:00。
- 本月：当月 1 日 00:00 至下月 1 日 00:00。
- 接口返回完整自然范围的每日序列；未来日期保持 0，但不参与连续天数计算。
- 分别计算任务、番茄和任一活动的“当前连续”和“范围内最长连续”。当前连续必须一直延续到今天；今天没有记录时为 0，不用昨天的记录猜测仍在连续。

### 可追溯与空数据

- 接口同时返回每日序列、汇总指标、连续天数和所选范围来源记录。
- 页面显示数据源、刷新时间、电脑本地时区、字段缺口和可展开来源明细。
- 没有分类或记录时显示真实的 0/空状态，不生成随机完成数、随机番茄或模拟连续天数。

## 备选方案

### 只统计三大功课分类

当前真实备份没有这些分类，会让已有任务完成记录全部不可见，也会把界面命名误当成 LifeUp 固定数据结构。未采用。

### 没有完成时间时用当前任务状态补到今天

可以让热力图看起来更完整，但无法证明任务是哪一天完成，会制造错误连续天数。未采用。

### `lasttime` 始终按分钟或始终按毫秒

不同接口存在两种单位。固定单一单位会让某些记录变成数万分钟或接近 0 分钟。采用现有兼容阈值，并用固定夹具覆盖两种格式。

## 影响

- 同一数据源、参考时间和电脑时区会得到稳定结果。
- 日常任务列表与热力图统计范围有意分开：列表保留三大功课产品结构，热力图反映全部真实完成活动。
- 字段不完整的旧版本数据可能显示缺口而不是“漂亮”的数字，这是保证真实性的取舍。
- 如果官方接口以后提供明确的单位或完成日志，应新增 ADR 或替代本兼容规则，而不是静默改变历史口径。
