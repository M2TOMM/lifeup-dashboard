# LifeUp Dashboard 产品路线图

版本：2026-07-18

## 一句话方向

把 LifeUp Dashboard 完善成一个安全、可靠、适合批量操作的电脑端控制台：本地备份负责完整管理，手机云人升负责实时查看和安全新增，两种数据源始终隔离。

## 当前产品基线

目前已经具备：

- 本地 LifeUp 备份 ZIP 的浏览器导入、工作副本读取、任务/商品/成就等管理与导出。
- 手机云人升连接、实时读取、只读浏览和预览确认后的新增任务。
- 任务、商品、成就、背包、合成、卡池、经济、历史等管理页面。
- 数据源隔离、Token 仅内存保存、云端预览令牌和幂等执行、基础 XSS 防护。
- 任务、商品、自定义成就和图标均已接入统一预览、快照、事务执行和逐行报告。
- “大道宏愿”“道行复盘”“日常功课”和“闭关修炼”已接入真实记录、来源明细和明确的数据缺口提示。
- 168 项自动化回归测试，当前全部通过。

仍然存在两类明显短板：

1. **云端体验仍可提升**：连接错误分类、慢接口渐进加载和实时分类/技能选择还不完整。
2. **交付体验不足**：README、启动说明、工作区清理和桌面版发布流程还没有跟上当前产品。

## 产品原则

- **数据安全优先**：任何新功能都不能以覆盖原始备份或直接改写手机已有数据为代价。
- **先预览，再执行**：批量写入必须显示将修改什么、错误在哪里、执行后结果如何。
- **工作副本优先**：本地管理只作用于工作副本，导出时生成新 ZIP，并验证可恢复性。
- **双数据源隔离**：`local` 和 `cloud` 不自动切换、不静默混合、不做双向同步。
- **真实数据优先**：没有真实字段支持的页面要明确标注演示，不能让用户误以为是 LifeUp 实时数据。
- **小步交付**：每个任务控制在一个专注开发会话内，完成后立即测试和浏览器验证。
- **暂不重写技术栈**：继续使用 Flask + 原生 HTML/JavaScript，先解决产品价值和可靠性，不急于迁移 React 或拆微服务。

## 阶段 0：可信基础（最高优先级）

### 0.1 备份 ZIP 安全校验

- 导入前检查 ZIP 结构、数据库文件、路径穿越、体积和解压后大小。
- 无效备份必须在替换当前工作区之前被拒绝。
- 错误提示说明“哪里不符合要求”和“用户下一步怎么做”。

### 0.2 原子导出与恢复验证

- 先写临时文件，验证 ZIP 和 SQLite 完整性后再生成最终导出文件。
- 浏览器流程拒绝把导出目标指向原始备份。
- 导出后自动显示文件路径、大小、校验结果和生成时间。

### 0.3 真正的快照系统

- [x] 将当前只保存路径的“快照”改成真实 ZIP 副本。
- [x] 快照存放在 `workspaces/snapshots/`，包含名称、来源、时间、大小和校验值。
- [x] 恢复快照时先创建新的工作副本，不直接覆盖当前文件。

### 0.4 批量接口严格校验

- 任务和商品批量接口只接受明确允许的操作。
- ID、价格、状态等参数必须在写数据库前完成校验。
- 无效请求返回错误，不能出现“没有修改却提示成功”。

### 0.5 核心流程回归测试

- 覆盖任务、商品、成就的新增、编辑、删除和批量操作。
- 覆盖打开工作副本、导出、重新打开导出文件的完整闭环。
- 每次测试后确认工作副本恢复、原始备份修改时间不变。

阶段验收：用户可以在工作副本中修改数据、创建快照、导出新备份、重新打开验证，并且任何失败都不会破坏此前可用的文件。

## 阶段 1：批量管理工作台（核心产品价值）

### 1.1 统一批量处理框架

所有批量功能共用四步：

1. 选择文件或粘贴数据。
2. 解析并显示预览表格。
3. 标出错误、重复项和将发生的改动。
4. 创建快照后执行，返回逐行结果报告。

### 1.2 任务批量管理（已完成）

- [x] 提供 UTF-8 BOM CSV/JSON 模板下载。
- [x] 支持新增任务的标题、分类、技能、金币/经验、频率、目标次数和冻结状态。
- [x] 按规范化标题识别已有任务和同一文件内重复项，由用户逐行选择跳过或新增。
- [x] 执行前自动快照，全部写入位于一个 SQLite 事务，失败整批回滚。

### 1.3 商品批量管理（已完成）

- [x] 提供 UTF-8 BOM CSV/JSON 模板，支持名称、分类、价格、库存、购买状态和 coin/exp 基础效果。
- [x] 同名商品由用户逐行选择跳过或仍然新增；新增商品同时建立商店与背包库存记录。
- [x] 批量改价提供固定值、增加/减少、百分比三种方式和修改前后对比。
- [x] 改价只更新 `shopitemmodel.price`，保留未知 `extrainfo`、购买限制、复杂效果、合成关系和库存记录。
- [x] 执行前自动快照，全部写入位于一个 SQLite 事务，价格在预览后变化时整批回滚。

### 1.4 成就批量管理（已完成）

- [x] 支持自定义成就新增、金币/经验奖励、分类、描述和安全图标引用。
- [x] 系统成就继续只读，同名冲突不能通过重复策略绕过。
- [x] 非空条件结构整行阻止并说明原因，本轮不写 `unlockconditionmodel`。
- [x] 执行复用一次性预览 token、digest、自动快照和单事务回滚；cloud 全链路拒绝。

### 1.5 图标资源管理器（已完成）

- [x] 浏览、搜索、筛选和分页查看工作副本图片，显示直接引用、无直接引用、失效引用、无法识别内容和扩展名不一致。
- [x] 安全上传 PNG/JPEG/GIF/WebP 到工作副本，限制 5 MiB，使用内容摘要文件名且不覆盖同名文件。
- [x] 商品和自定义成就支持批量替换；技能和系统成就保持只读。
- [x] 替换复用一次性预览 token、digest、自动快照和单事务回滚。
- [x] 导出前验证所有直接本地图标引用；缺失或内容无法识别时阻止生成 ZIP。

阶段验收：用户可以用电脑一次处理几十条任务、商品或成就，并在执行前看懂变化、执行后拿到报告、出现问题时恢复快照。

## 阶段 5：AI 与 MCP 接入

### 5.1 LifeUp MCP 适配层（Task 17，已完成）

- MCP 只做协议转换并调用现有 Flask HTTP API，不直接访问 SQLite、备份 ZIP 或本地映射文件。
- 第一版提供状态、任务、商品、成就、仪表盘、专注和宏愿等只读工具。
- 新增任务复用现有“预览令牌 → 用户明确确认 → 幂等执行”流程，不开放编辑、删除和批量写入。
- 每次调用明确指定 `local` 或 `cloud`，不自动切换、不混合数据源。
- Token 只保留在现有服务进程内存中，不进入 MCP 配置、响应或日志。
- 提供 Windows PowerShell 启动命令以及 Codex/Claude 客户端配置和实际调用验收。

阶段验收：MCP 返回结果与现有 HTTP API 一致，AI 客户端能够完成只读查询和安全新增任务，同时原始备份、手机已有数据与 Token 不受影响。

## 暂时不做

- “LifeUp 云备份成长伴侣”作为独立项目推进，不纳入本 Dashboard 路线图或代码库。
- 不自动同步手机和本地备份。
- 不通过 `/data/import` 写手机。
- 不直接修改手机 SQLite。
- 不编辑或删除手机已有任务、商品和成就。
- 不在没有官方安全接口和独立风险评估前扩展云端写入范围。
- 不为了“技术看起来更现代”重写整个前端框架。
